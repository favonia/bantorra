Library Loading

A library is identified by a JSON file in its root directory, which is called "anchor".

load_library_from_root manager lib_root loads the library at the directory lib_root from the file system. It is assumed that there is an anchor file is right at lib_root.

parameter manager

The library manager.

parameter lib_root

The root of the library, which should be a directory.

returns

The loaded library.

load_library_from_dir manager dir assumes the directory dir resides in some library and will try to find the root of the library by locating the anchor file. It then loads the library marked by the anchor.

parameter dir

A directory that is assumed to be inside some library.

returns

val load_library_from_unit : t -> BantorraBasis.File.filepath -> suffix:string -> (library * unitpath option, [> `InvalidLibrary of string ]) Stdlib.result

locate_anchor_from_unit filepath ~suffix assumes filepath ends with suffix and the file at filepath resides in some library. It will try to find the root of the library and load the library.

parameter manager

The library manager.

parameter filepath

The corresponding file path.

parameter suffix

The suffix of the unit on the file system. Note that the dot is included in the suffix---the suffix of file.ml is .ml, not ml.

returns

The loaded library and the unit path in the library. The unit path is None if the file is actually inaccessible, probably due to another mounted library shadowing the unit.

Composite Resolver

These functions will automatically load the dependencies.

resolve manager lib unitpath ~suffix resolves unitpath in the library in the library lib and returns the eventual library where the unit belongs and the corresponding file path with the specified suffix.

parameter manager

The library manager.

parameter max_depth

Maximum depth for resolving recursive library mounting. The default value is 100.

parameter lib

The library.

parameter unitpath

The path to be resolved.

parameter suffix

The suffix shared by all the units in the file system.
