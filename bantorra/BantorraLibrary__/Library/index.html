Hooks for Library Management

The following API is for a library manager to chain all the libraries together. Please use the high-level API in BantorraManager.Manager instead.

val to_filepath : global:(cur_root:string -> BantorraLibrary.Anchor.lib_ref -> unitpath -> suffix:string -> string) -> t -> unitpath -> suffix:string -> string

to_filepath ~global lib unitpath ~suffix turns a unit path into a file path appended with suffix.

parameter global

The global resolver for unit paths pointing to other libraries.

val replace_cache : global:(cur_root:string -> BantorraLibrary.Anchor.lib_ref -> unitpath -> source_digest:Stdlib.Digest.t -> BantorraBasis.Marshal.t -> Stdlib.Digest.t) -> t -> unitpath -> source_digest:Stdlib.Digest.t -> BantorraBasis.Marshal.t -> Stdlib.Digest.t

replace_cache ~global lib unitpath ~source_digest value replaces the cache associated with unitpath and source_digest with value. It returns the digest of the stored cache.

parameter global

The global cache replacer for unit paths pointing to other libraries.

parameter source_digest

The digest of the source file. For example, .agda files in the Agda system.

val find_cache_opt : global:(cur_root:string -> BantorraLibrary.Anchor.lib_ref -> unitpath -> source_digest:Stdlib.Digest.t -> cache_digest:Stdlib.Digest.t option -> BantorraBasis.Marshal.t option) -> t -> unitpath -> source_digest:Stdlib.Digest.t -> cache_digest:Stdlib.Digest.t option -> BantorraBasis.Marshal.t option

find_cache ~global lib unitpath ~source_digest ~cache_digest value finds the cache associated with unitpath and source_digest. If cache_digest is None, it means the digest checking is skipped. One should use the digest returned by replace_cache whenever possible.

parameter global

The global cache finder for unit paths pointing to other libraries.

parameter source_digest

The digest of the source file. For example, .agda files in the Agda system.

parameter cache_digest

The digest of the stored cache. For example, .agdai files in the Agda system.
