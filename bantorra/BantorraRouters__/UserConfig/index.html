A router that uses per-user configuration files.

Argument Format

The router takes a JSON argument in one of the following formats:

{
  "name": "bantorra",
  "version": "0.1.0"
}
{
  "name": "bantorra",
  "version": null
}
{ "name": "bantorra" }

A missing version is understood as the version null (different from the string "null"). Therefore, the last two specifications are identical. Versions are compared using structural equality. There is no smart comparison, version globbing, or ordering between versions. Each version is completely independent of each other, and the version null is distinct from any string versions.

Configuration Format

By default, the configuration is at $XDG_CONFIG_HOME/${app_name}/${config}. The exact path is given by BantorraBasis.File.get_xdg_config_home concatenated with the argument config given to router.

Here is an example configuration file:

{
  "format": "1.0.0",
  "libraries": [
    {
      "name": "num",
      "version": "3",
      "at": "/usr/lib/something/num34"
    },
    {
      "name": "tcp",
      "version": "2",
      "at": "/usr/lib/something/tcp21"
    }
  ]
}

You can register multiple versions of the same library in the user configuration file, as long as incompatible versions are not in use at the same time. Two versions are compatible if they refer to the same location in the file system. (For example, if both versions "1.0" and "1" point to /usr/lib/hello/1.0, then they are compatible and can be used interchangeably.) Any attempt to load incompatible versions during the program execution would abort the library routing.

{
  "format": "1.0.0",
  "libraries": [
    {
      "name": "tcp",
      "version": "2",
      "at": "/usr/lib/something/tcp21"
    },
    {
      "name": "tcp",
      "version": "2.1",
      "at": "/usr/lib/something/tcp21"
    }
  ]
}
{
  "format": "1.0.0",
  "libraries": [
    {
      "name": "tcp",
      "versions": ["2", "2.1"],
      "at": "/usr/lib/something/tcp21"
    }
  ]
}

The following is a more complicated example:

{
  "format": "1.0.0",
  "libraries": [
    {
      "name": "num",
      "versions": ["2", "2.5"],
      "at": "/usr/lib/something/num25"
    },
    {
      "name": "num",
      "version": "3",
      "at": "/usr/lib/something/num34"
    },
    {
      "name": "num",
      "version": ["3.4", null],
      "at": "/usr/lib/something/num34"
    },
    {
      "name": "tcp",
      "version": "2",
      "at": "/usr/lib/something/tcp21"
    },
    {
      "name": "tcp",
      "version": "2.1",
      "at": "/usr/lib/something/tcp21"
    }
  ]
}

Given the above configuration file, versions "2" and "2.5" of the library num are compatible because they point to the same location in the file system. Versions null and "2" of the library "num" are not compatible because they point to different locations, and thus they cannot be used at the same time. Note that there is no version null registered for the library tcp, so one has to specify a string version (either "2" or "2.1") for it, or the routing would fail. The philosophy is that all entries must be explicitly listed.

Builder

val router : ?⁠xdg_macos_as_linux:bool -> app_name:string -> config:string -> Bantorra.Router.t

router ?xdg_as_linux ~app_name ~config constructs a router that reads the user configuration. The location of the user configuration is given by BantorraBasis.File.get_xdg_config_home?as_linux:xdg_as_linux concatenated with config.

If the configuration file does not exist, an empty mapping is used, which means the router would reject every request.

parameter xdg_as_linux

Whether the XDG path construction should follow the Linux convention, ignoring the OS detection.

parameter app_name

The application name for generating a suitable directory to put the configuration file.

parameter config

The file name of the configuration file.

val clear_cached_configs : unit -> unit

Configuration files are all cached to reduce I/O load, but perhaps you are bypassing this module to modify them. In that case, one should call this function to force rereading configuration files.
