make ?fast_checker ?args_dumper router creates a new router that can be used in Manager.init.

parameter router

router ~starting_dir args is responsible for finding the root of the library specified by args. A library manager will feed the router with unparsed router_arguments field in an anchor file. (See Manager.) The return value is None if the resolution fails.

parameter fast_checker

A validity checker for dependencies in anchor files when loading a library (before the units from other libraries are actually needed). It is taking the same arguments as the router does, but it only needs to predict whether the resolution would be successful. Some library resolution could be expensive (e.g., involving downloading the sources from the server) and a faster, less precise validity checker might be desired. If absent, full resolution will be used as the checker.

parameter args_dumper

A hack to dump the arguments as a string for ugly-printing. This will be replaced by proper pretty-printing in the future. If absent, BantorraBasis.Marshal.dump is used.
